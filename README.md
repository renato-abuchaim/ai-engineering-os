Um sistema modular de copilotos especializados para:

* QA;
* automação;
* debugging;
* desenvolvimento;
* arquitetura;
* refatoração;
* aprendizado técnico;
* e engenharia de software.

A ideia do projeto é simples:

> O operador é o orquestrador.
> Os modos são copilotos especializados.

Cada modo possui:

* responsabilidade única;
* comportamento próprio;
* profundidade técnica específica;
* e foco operacional claro.

---

# 🎯 Objetivo do Projeto

Este projeto foi criado para transformar IA em um sistema realmente útil para:

* estudar tecnologia;
* planejar implementações;
* diagnosticar problemas;
* gerar código;
* investigar bugs;
* melhorar arquitetura;
* estruturar testes;
* e evoluir tecnicamente.

Ao invés de usar prompts genéricos, o sistema separa responsabilidades em modos especializados.

---

# 🧩 Modos do Sistema

| Mode             | Função Principal                             |
| ---------------- | -------------------------------------------- |
| 📚 STUDY MODE    | Aprendizado técnico e entendimento profundo  |
| ❓ ASK MODE       | Perguntas, análise e diagnóstico rápido      |
| 🧠 PLAN MODE     | Planejamento técnico antes da execução       |
| 🤖 AGENT MODE    | Implementação e execução                     |
| 🐛 DEBUG MODE    | Investigação e causa raiz                    |
| ♻️ REFACTOR MODE | Evolução estrutural e melhoria de código     |
| 🧪 QA TEST MODE  | Estratégia, cobertura e engenharia de testes |

---

# 🚀 Como Usar

1. Escolha o modo mais adequado para a tarefa.
2. Copie o conteúdo do arquivo do modo desejado.
3. Cole no seu copiloto de IA.
4. Descreva seu problema, dúvida ou objetivo.
5. Atualize os modos conforme sua evolução técnica.

---

# 📌 Exemplos Rápidos

* Quer estudar Playwright → `📚 STUDY MODE`
* Quer entender um erro rapidamente → `❓ ASK MODE`
* Quer planejar uma automação → `🧠 PLAN MODE`
* Quer implementar código → `🤖 AGENT MODE`
* Quer investigar um bug → `🐛 DEBUG MODE`
* Quer melhorar arquitetura ou limpar código → `♻️ REFACTOR MODE`
* Quer estruturar cenários de teste → `🧪 QA TEST MODE`

---

# 📚 STUDY MODE

O modo **Study** é focado em aprendizado técnico profundo.

Ele ajuda a:

* entender conceitos;
* aprender QA;
* estudar arquitetura;
* desenvolver raciocínio técnico;
* compreender trade-offs;
* aprender debugging;
* aprender automação.

Funciona como um tutor técnico progressivo.

📄 Arquivo: `01-STUDY-MODE.md`

---

# ❓ ASK MODE

O modo **Ask** é usado para:

* perguntas rápidas;
* análise de erros;
* dúvidas técnicas;
* comparações;
* diagnósticos simples;
* explicações rápidas.

Ele atua como um copiloto de suporte técnico.

📄 Arquivo: `02-ASK-MODE.md`

---

# 🧠 PLAN MODE

O modo **Plan** é responsável por:

* estruturar soluções;
* planejar implementações;
* prever riscos;
* organizar execução;
* definir estratégia;
* mapear impacto técnico.

Ele pensa antes da implementação.

📄 Arquivo: `03-PLAN-MODE.md`

---

# 🤖 AGENT MODE

O modo **Agent** é o executor técnico.

Ele é responsável por:

* implementar código;
* gerar estruturas;
* criar automações;
* modificar múltiplos arquivos;
* integrar funcionalidades;
* executar tarefas técnicas complexas.

📄 Arquivo: `04-AGENT-MODE.md`

---

# 🐛 DEBUG MODE

O modo **Debug** é especializado em:

* investigação de bugs;
* troubleshooting;
* causa raiz;
* análise de logs;
* testes flaky;
* problemas intermitentes;
* observabilidade.

O foco principal é:

* separar sintoma de causa.

📄 Arquivo: `05-DEBUG-MODE.md`

---

# ♻️ REFACTOR MODE

O modo **Refactor** melhora:

* arquitetura;
* legibilidade;
* organização;
* modularização;
* manutenção;
* qualidade estrutural.

O objetivo é:

* reduzir dívida técnica;
* melhorar qualidade sem alterar comportamento esperado.

📄 Arquivo: `06-REFACTOR-MODE.md`

## ⚠️ O que NÃO deve ser alterado drasticamente

Cada modo possui uma identidade cognitiva própria.

Exemplos:

* 📚 STUDY MODE → didático, progressivo e focado em aprendizado
* ❓ ASK MODE → consultivo, objetivo e focado em respostas rápidas
* 🧠 PLAN MODE → estratégico, organizado e focado em planejamento
* 🤖 AGENT MODE → executor, prático e focado em implementação
* 🐛 DEBUG MODE → investigativo, analítico e focado em causa raiz
* ♻️ REFACTOR MODE → arquitetural e focado em qualidade estrutural
* 🧪 QA TEST MODE → analítico, preventivo e focado em qualidade e cobertura

A recomendação é:

* ajustar o estilo;
* ajustar o nível técnico;
* ajustar a profundidade das respostas;
* ajustar exemplos e explicações;

Sem destruir a função principal do modo.

Isso mantém:

* consistência;
* especialização;
* previsibilidade;
* qualidade das respostas;
* separação clara de responsabilidades.

## 🧠 Filosofia do Sistema

Os modos foram projetados para evoluir junto com o operador.

Conforme o conhecimento técnico aumenta:

* a stack evolui;
* as áreas de estudo evoluem;
* e a personalidade dos agentes pode evoluir também.

O objetivo é criar copilotos cada vez mais alinhados ao seu fluxo real de engenharia e QA.
